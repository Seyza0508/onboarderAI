# Northstar Payments Engineering Onboarding Handbook

## Welcome
Welcome to Northstar Payments Engineering. This handbook helps new engineers become productive in the first two weeks.

## Onboarding Principles
- Start with security and access.
- Prioritize development environment setup.
- Learn team architecture before making production changes.
- Ask for help early and document blockers.

## Week 1 Outcomes
- Company accounts are fully activated.
- Local development environment is running.
- Team documentation is reviewed.
- First non-critical pull request is opened.

## Standard Setup Sequence
1. Activate SSO and MFA.
2. Join Slack channels: `#eng-onboarding`, `#payments-dev`, `#incident-updates`.
3. Set up VPN and verify internal wiki access.
4. Accept GitHub organization invite and confirm repo permissions.
5. Install Docker, Python, Node.js, and package managers.
6. Clone required repositories and run local services.

## Required Reading
- `payments_architecture_overview.md`
- `incident_runbook.md`
- `troubleshooting_faq.md`
- Team-specific onboarding guide

## First Task Guidance
- Choose a low-risk task tagged `good-first-task`.
- Confirm acceptance criteria with your mentor.
- Run tests locally before opening a PR.

## Escalation Expectations
Escalate after 24 hours for access blockers and after 2 failed troubleshooting attempts for environment blockers.
