# Payments Team Onboarding

## Team Mission
Payments builds and maintains checkout, authorization, settlement, and reconciliation systems.

## First Week Priorities
- Access payments repos and dashboards.
- Understand core services and event flow.
- Complete sandbox transaction walkthrough.

## Required Repositories
- `payments-api`
- `payments-worker`
- `payments-web`

## Key Meetings
- Team standup (daily)
- Onboarding sync with mentor (twice weekly)
- Architecture walkthrough (week 1)

## Done Criteria For Week 1
- Local services run successfully.
- One docs improvement PR submitted.
- One non-critical bug triaged and discussed.

## Recommended Learning Order
1. `payments_architecture_overview.md`
2. `incident_runbook.md`
3. `troubleshooting_faq.md`
